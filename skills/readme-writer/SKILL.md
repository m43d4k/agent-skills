# README Writer

完全性を求めて推測や過剰な追加をせず、確認できる範囲で妥当な初稿を作る。その後の修正でも同じ方向性を維持する。

## 基本方針

- リポジトリ固有の情報だけを書く。
- ソース、設定、スクリプト、CI、既存ドキュメントから確認できない内容は書かない。
- 未実装の機能、予定、推測、一般的な推奨事項を追加しない。
- 一般的なテンプレートより、リポジトリ固有の情報と既存の文脈を優先する。
- 必要のないセクションを機械的に追加しない。
- 必要な情報を削って短くせず、重複と冗長な表現を減らす。
- 箇条書きで明確に表現できる内容は箇条書きにする。
- 箇条書きの各項目では、一つの情報だけを扱う。

## 調査

- 実装コード、設定ファイル、パッケージ設定、ビルド設定、CI、実行スクリプト、既存ドキュメントを確認する。
- 実際の動作を規定するファイルを優先する。
- READMEに記載する対象エージェントや対応プラットフォームは、リポジトリ内の実装、設定、既存ドキュメントから判断する。
- README作成に使用しているAIエージェントや実行環境を、それらの判断根拠にしない。
- 既存READMEの記述を、他の根拠なく拡張したり具体化したりしない。
- package manager、必要環境、コマンド、対応OSなどを一般論から推測しない。
- 確認できない内容は断定しない。

## 概要文

- READMEを開いた人が、短時間で「何をするものか」を把握できるように書く。
- 実装方式ではなく、人が何をするために使うものかを利用者の視点で書く。
- 詳細な機能説明、操作手順、技術構成は書かない。
- 機能一覧を文章へ変換せず、全体として何をするものかを書く。
- 必要な情報を無理に一文へ詰め込まない。
- 複数の文に分ける場合、1文目では個別機能を列挙せず、全体の用途を示す。
- 2文目以降で、対象、特徴的な操作、制約、既存手段との差分を補足する。
- 概要文だけで詳細を説明し切ろうとしない。
- リポジトリ固有の価値を、形容詞ではなく用途、対象、制約、差分で示す。
- 実装から確認できない効果や改善量を主張しない。
- H1にプロジェクト名がある場合、直後の文を同じプロジェクト名から始めない。

## 構造

- 修正時は文言より先に、情報の順序、セクション分割、重複、配置を確認する。
- 構造上の問題を、言い換えだけで解決しない。
- 読者が最初に探す情報を上に置く。
- セクションの順序を固定しない。
- 既存READMEがある場合は、用語、見出し階層、説明順序を尊重する。
- 必要がない限り、既存の構成を全面的に変更しない。
- 同じ事実を複数のセクションで言い換えて繰り返さない。

## 言語と文体

- 本文と見出しは日本語で書く。
- 文体は常体で統一する。
- コマンド、コード、API名、製品名、UI上の表記は原文を維持する。
- 宣伝的な形容詞や評価語を使わない。
- 読者への呼びかけ、疑問文、煽り文句を使わない。
- 既存手段や競合を必要なく批判しない。
- 見出しや文脈から明らかな主語を、「このツール」「本リポジトリ」などで繰り返さない。
- セクション見出しに含まれる語を、本文冒頭で機械的に言い直さない。

## コマンド

- 初回準備、通常実行、ビルド、テストのコマンドを役割に応じて分ける。
- コマンドは、設定やスクリプトと一致する再現可能な形で書く。
- 利用者向けコマンドと開発者向けコマンドを分ける。
- 読者が最初に探すコマンドを上に置く。
- CLIやコマンド中心のツールでは、主要な利用コマンドを概要の直後に置く。
- 実行確認していない結果を、確認済みの事実として断定しない。
- package managerやコマンドを一般論で置き換えない。

## READMEとdocs

- 基本的なセットアップと主要操作はREADME内で完結させる。
- 設計、内部構造、詳細仕様、長いトラブルシューティングはREADMEに詰め込まず、既存の詳細文書へ分離する。
- 新しい詳細文書は、ユーザーから指示された場合だけ作成する。
- 詳細文書が存在する場合だけリンクを追加する。

## 作者情報

- 作者または管理主体を確認できる場合はREADMEに記載する。
- 既存README、LICENSE、パッケージ設定、リモートURLなどから確認する。
- Gitのコミット作成者だけを根拠に、作者と断定しない。
- リモートURLの所有者を、根拠なく作者と断定しない。
- 作者を確認できない場合は、確認できる管理主体だけを記載する。
- 個人リポジトリでは、名前またはハンドルと公開プロフィールへのリンクを書く。
- 組織管理のリポジトリでは、組織名または管理主体を書く。
- 作者とコントリビューターを混同しない。
- 公開されていない連絡先を追加しない。
- 原則としてREADME末尾に置く。

## 適用範囲

- この方針を日本語READMEの新規作成、修正、追記、構成変更に適用する。
- 追加指示で変更されていない方針は維持する。
