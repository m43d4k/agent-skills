# style-guide v1 / v2 箇条書き入力比較

## 結論

3題材すべてで、v2は共通入力にある事実、観測、判断、留保を維持した。入力にない評価変更や体験は追加せず、技術的な意味の強化も確認されなかった。

Blade EQとLoudness Proでは、v2の方が試行、観測、評価変更、適用判断の接続が明確だった。評価変更のないVX-Qでも、v2は架空の使用前予想や物語を追加せず、観測と判断を近接させた。

今回設定した正式採用の条件を満たしているため、`references/style-guide-v2.md`を現行版として採用できると評価する。この比較ではstyle guide自体の置き換えは実施していない。

## 比較方法

各サンプルから、参考著者の文体を除去した共通入力を作成した。共通入力では、次の情報を分けて記録している。

- 使用条件と試行
- 使用前の認識
- 観測
- 判断と判断履歴
- 制約、留保、未確認事項

各共通入力だけを材料として、`references/style-guide.md`を適用したv1と、`references/style-guide-v2.md`を適用したv2を作成した。両稿は独立して読める記事の一部分とし、同じ情報量と同程度の長さにそろえた。

評価では、共通入力との対応を先に確認し、元サンプルは共通入力の抽出内容を確認するために使用した。

## 1. Blade EQ

### 確認対象

- SonicEQに近いという予想
- ブーストとカットで確認した音の傾向
- PEQ3000を想起させるという試用後の評価
- メインEQとしての用途判断
- 聴感評価であり、同じ設計や同じ音を意味しないという留保

### 出力差

v1も使用前と試用後の評価を残しているが、冒頭で評価変更を示した後、音の観測を別の段落で説明している。そのため、どの観測を経て評価が変わったのかは、読者が前後を対応させる必要がある。

v2は、使用前の予想、ブーストとカットで得た観測、試用後にPEQ3000を想起したという順序を一つの段落内に置いた。評価変更を大きな反転として扱わず、聴感上の小さな修正として保存している。

### 評価

- 判断履歴: v2が明確
- 入力外の物語: なし
- 定型表現: 「評価へ変わった」は1回。機械的な反復なし
- 意味の強化: なし。「想起させる」を同一設計や同一音へ変更していない
- 留保: 両稿とも保存

判定はv2優位とする。

## 2. Loudness Pro

### 確認対象

- 初見での正確さと手軽さへの肯定的評価
- 操作を続けた後に加わった表現上の限界
- 手作業で設計した結果との比較
- プリリミッターという条件付き用途
- カジュアル用途での時間短縮
- 提案値が表示されない制約

### 出力差

v1は、Realtime AIの利点、表現上の制約、適用用途をセクションごとに整理している。情報は保存されているが、初見の評価が継続使用によってどのように限定されたかは、複数のセクションに分かれている。

v2は、初見の観測と評価を先に置き、継続使用と複数の書き出し比較によって別の評価軸が加わったことを直後に示した。そのうえで、肯定的評価を撤回せず、仕事ではプリリミッターとして使うという適用判断へ接続している。

### 評価

- 判断履歴: v2が明確
- 入力外の物語: なし。肯定から否定への単純な反転を作っていない
- 定型表現: 評価を示す語の集中や同一文型の反復なし
- 意味の強化: なし。「正確」「人間には困難」を筆者の評価として限定
- 制約と比較結果: 両稿とも保存

判定はv2優位とする。

## 3. VX-Q

### 確認対象

- 入力に使用前の予想が存在しない
- 4バンド構成とSATモードによる作業上の観測
- ローシェルフ、MODE S、MODE Zの用途判断
- ディエッサー相当機能の不在とVARIの問題
- MODE Sの由来が未確認であること

### 出力差

v1は機能単位で観測と判断を整理している。v2は、4バンド構成による迷いの減少と初期作業での用途判断、ローシェルフの観測と評価、MODE SとZの試用結果と選好をそれぞれ近接させている。

重要なのは、v2が使用前の予想、評価の反転、偶発的な発見を追加しなかった点である。判断履歴が存在しない箇所では、試した内容と得られた判断だけを記述している。

### 評価

- 判断履歴: 存在する範囲だけを両稿とも保存。v2は観測と判断の対応がやや明確
- 入力外の物語: なし
- 定型表現: 「評価した」「判断した」は各1回で、異なる根拠に対応している
- 意味の強化: なし
- 失敗、制約、未確認事項: 両稿とも保存

判定はv2がわずかに優位とする。

## v2で改善した点

- 事前予想、試行、観測、評価変更を同じ流れの中で読める。
- 肯定的評価と否定的評価を単純な反転にせず、評価軸と適用条件の違いとして残せる。
- 観測から生じた用途判断を、根拠の近くへ置ける。
- 判断履歴がない材料では、使用前の認識や発見を新しく作らない。

## v2で残る傾向

v2はv1よりも「評価」「判断」に相当する語を明示しやすい。今回の出力では同一表現の連発には至らず、各語も異なる根拠に対応しているため、許容範囲と判断した。

今後の記事で、各セクションが「観測した」「評価した」「判断した」という同じ終わり方になる場合は、判断履歴そのものではなく文型が目立っていないかを確認する必要がある。これは正式採用を妨げる問題ではなく、運用時の確認事項とする。

## 正式採用判定

事前に設定した不合格条件は、3題材のいずれにも発生しなかった。

- 入力にない体験、感情、予想、発見の追加: なし
- 判断に影響する失敗、条件、未確認事項の削除: なし
- 「異なる」を「逆」、「関連」を「原因」へ変える意味強化: なし
- 存在しない評価変更の追加: なし

また、判断履歴がある2題材ではv2が因果関係を明確にし、判断履歴がない題材でも余計な物語を作らなかった。したがって、今回の比較基準では`references/style-guide-v2.md`を現行版として採用できる。
