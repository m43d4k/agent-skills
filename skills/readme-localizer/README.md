# readme-localizer

日本語READMEを原文として、GitHub向けの英語READMEを作成・更新するスキルである。

翻訳によって内容や構成を作り変えず、日本語版との意味と技術情報の整合性を維持する。

## 対象ファイル

二言語化後は、英語版を `README.md`、日本語版を `README.ja.md` とする。他のファイルは、明示的に指定された場合だけ翻訳する。

## 処理方針

日本語の `README.md` だけが存在する場合は、内容を確認して `README.ja.md` へリネームし、それを原文として英語版の `README.md` を作成する。両方のREADMEには相互の言語切り替えリンクを追加する。

すでに両言語のREADMEが存在する場合は、`README.ja.md` の変更を英語版へ反映する。意味のある差分について正しい内容を判断できない場合は、推測で統合しない。

## 翻訳方針

- 原文の情報を追加、削除、要約しない。
- 見出し階層、情報の順序、段落や箇条書きの対応関係を維持する。
- 逐語訳より、技術文書として自然で正確な英語を優先する。
- コマンド、コード、ファイルパス、URL、API名、製品名、バージョン番号を変更しない。
- 原文にない利点、評価、推測、補足を追加しない。
- 作者情報が存在する場合は、その内容を維持する。

## 対象外

- READMEの内容や構造の再設計
- 日本語版への新しい説明の追加
- `docs/` 以下の自動翻訳
- ソースコード内の文字列の翻訳
- 翻訳と無関係なファイルの変更
