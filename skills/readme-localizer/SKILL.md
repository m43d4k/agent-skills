# README Localizer

日本語READMEを原文として、GitHub向けの自然な英語READMEを作成・更新する。

翻訳によって内容や構成を作り変えず、日本語版との意味と技術情報の整合性を維持する。

## 基本方針

- 日本語版を原文として扱う。
- 英語版は日本語版と同じ情報を伝える。
- 原文の情報を追加、削除、要約しない。
- READMEの構成改善と翻訳を同時に行わない。
- 逐語訳より、技術文書として自然で正確な英語を優先する。
- 原文にない利点、評価、推測、補足を追加しない。
- 宣伝的、誇張的、会話的な表現へ変更しない。
- 曖昧な原文を推測によって具体化しない。

## 対象ファイル

二言語化後のファイル構成は次のとおりとする。

```text
README.md
README.ja.md
```

- `README.md` を英語版とする。
- `README.ja.md` を日本語版とする。
- 他のファイルは、ユーザーから指示された場合だけ翻訳する。

## 初回変換

日本語の `README.md` だけが存在する場合は、次の手順で二言語化する。

1. `README.md` が日本語版であることを確認する。
2. `README.ja.md` が存在しないことを確認する。
3. 日本語の `README.md` を `README.ja.md` にリネームする。
4. `README.ja.md` を原文として、英語版の `README.md` を作成する。
5. 両方のREADMEに言語切り替えリンクを追加する。
6. 両言語の内容とMarkdownの整合性を確認する。

既存の `README.md` または `README.ja.md` を確認せずに上書きしない。

## 既存英語版の更新

`README.md` と `README.ja.md` が存在する場合は、次の方針で更新する。

- `README.ja.md` の変更を英語版へ反映する。
- 変更されていない箇所を必要なく書き換えない。
- 英語版だけに存在する情報を、自動的に日本語版へ取り込んだり削除したりしない。
- 両言語に意味のある差分がある場合は、リポジトリの実装や既存文書を確認する。
- どちらが正しいか判断できない差分を、推測で統合しない。
- ユーザーから英語版だけの修正を指示された場合は、その範囲だけを変更する。

## 言語切り替え

- H1の直後に言語切り替えリンクを置く。
- 英語版には日本語版へのリンクを置く。
- 日本語版には英語版へのリンクを置く。
- 既存の言語切り替え表記がある場合は、必要なく形式を変更しない。
- 同じ言語への自己リンクは追加しない。

英語版：

```markdown
[日本語](README.ja.md)
```

日本語版：

```markdown
[English](README.md)
```

## 翻訳

- 見出し階層と情報の順序を維持する。
- 段落、表、箇条書き、コードブロックの対応関係を維持する。
- 項目を理由なく統合または分割しない。
- 日本語の文構造をそのまま英語へ移さず、技術文書として自然な構文にする。
- 日本語で省略されている主語や目的語は、意味を変えない範囲で補う。
- 同じ用語はREADME内で一貫して訳す。
- GitHub上のソフトウェア文書で一般的な表現を使う。
- 不自然な直訳や過度に丁寧な表現を避ける。
- 日本語版が示す用途、対象、制約、差分を維持する。
- 原文より強い効果や価値を主張しない。
- 見出しの意味と階層を維持し、英語として簡潔な表現にする。
- 見出しへのアンカーリンクがある場合は、英訳後のアンカーと一致させる。

## 翻訳しない要素

次の要素は原則として変更しない。

- コマンド
- コード
- ファイル名
- ディレクトリ名
- ファイルパス
- URL
- Markdownリンクのリンク先
- 画像パス
- バッジ
- API名
- クラス名
- 関数名
- 変数名
- パッケージ名
- 製品名
- サービス名
- UI上の実際の表記
- キーボードショートカット
- バージョン番号
- HTML属性

リンクテキスト、画像の代替テキスト、コードブロック外の説明文は、意味に応じて英訳する。

UI上の文字列は実装されている表記を維持する。UIが日本語の場合、英語版で架空の英語表記を作らない。

## コマンドとコード

- コマンド、オプション、引数、出力例を変更しない。
- コードブロックの内容と言語指定を維持する。
- コードブロック内のコメントは、原文との対応が必要な場合だけ翻訳する。
- パッケージマネージャーや実行方法を一般論で変更しない。
- 翻訳だけを理由に、コマンドや技術仕様を修正しない。
- 実行確認していない結果を事実として追加しない。

## リンクと画像

- 相対リンクが各READMEの位置から有効か確認する。
- 言語切り替えリンクが相互に正しいか確認する。
- 画像パスを必要なく変更しない。
- 同じ画像を使用する場合は、両言語で同じパスを使う。
- 言語別の画像が存在する場合だけ、対応する画像へ変更する。
- 見出しアンカーを含むリンクは、英訳後の見出しと一致させる。
- 外部URLを翻訳や推測で変更しない。

## 作者とライセンス

- 作者名、Copyright holder、プロフィールURLを変更しない。
- ライセンス名と `LICENSE` へのリンクを維持する。
- 既存の作者情報とライセンス情報を削除しない。
- セクション見出しと説明文だけを必要に応じて英訳する。
- `LICENSE` ファイル自体は翻訳、修正、再生成しない。
- ライセンス年を翻訳作業の一環として更新しない。

## 確認

作業後に次を確認する。

- 両言語で情報、見出し構造、項目の対応が維持されているか。
- コマンド、コード、パス、URLが意図せず変更されていないか。
- 相対リンク、画像、見出しアンカーが有効か。
- 言語切り替えリンクが相互に正しいか。
- 作者情報とライセンス情報が維持されているか。
- 英語版に不要な日本語が残っていないか。
- 日本語版が翻訳作業によって不必要に変更されていないか。

## 対象外

- READMEの内容や構成の再設計
- 日本語版への新しい説明の追加
- `docs/` 以下の自動翻訳
- ソースコード内の文字列の翻訳
- `LICENSE` の変更または再生成
- 翻訳と無関係なファイルの変更

READMEの内容や構造を修正する必要がある場合は、先に日本語版を修正し、その後で英語版へ反映する。
