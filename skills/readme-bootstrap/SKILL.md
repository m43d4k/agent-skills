# README Bootstrap

リポジトリ作成直後に、タイトル、概要、作者、ライセンスだけを含む日本語READMEとMIT Licenseを作る。

## 既定値

- 作者名は `m43d4k` とする。
- 作者URLは `https://github.com/m43d4k` とする。
- Copyright holderは作者名と同じ値を使う。
- Copyright yearは `LICENSE` 作成時の西暦とする。

## 基本方針

- `README.md` が存在しない場合に使用する。
- READMEにはH1、概要文、作者情報、ライセンス情報だけを書く。
- リポジトリ直下にMIT Licenseの `LICENSE` を作成する。
- セットアップ、使い方、技術構成、機能一覧などを追加しない。
- 後から情報を追加する前提で、現時点で確認できる内容だけを書く。
- 本文と見出しは日本語で書く。
- 文体は常体で統一する。
- 宣伝的な形容詞、評価語、煽り文句を使わない。
- 確認できない情報を推測しない。

## タイトル

- リポジトリ名またはユーザーが指定した名称をH1にする。
- 製品名、パッケージ名、プロジェクト名が明確な場合は、その表記を優先する。
- 大文字と小文字、空白、記号などの表記を必要なく変更しない。

## 概要文

- 人が何をするためのリポジトリかを簡潔に書く。
- 実装方式より、利用者から見た用途を優先する。
- 個別機能を列挙せず、全体として何をするものかを示す。
- 必要な情報を無理に一文へ詰め込まない。
- H1直後の文を、同じプロジェクト名から始めない。
- ユーザーの依頼、リポジトリの説明、既存ファイルから確認できる内容を使う。
- 対象エージェントや対応プラットフォームは、ユーザーの依頼またはリポジトリ内の根拠から判断する。
- README作成に使用しているAIエージェントや実行環境を、その判断根拠にしない。
- リポジトリ名だけから用途を推測しない。

## 作者情報

- README末尾に作者情報を記載する。
- 作者名とリンク先には「既定値」の作者名と作者URLを使う。
- 公開されていない連絡先を追加しない。

## ライセンス

- `assets/MIT-LICENSE.txt` を、リポジトリ直下へ `LICENSE` という名前でコピーする。
- `LICENSE` ディレクトリは作成しない。
- `{{YEAR}}` を「既定値」のCopyright yearへ置き換える。
- `{{COPYRIGHT_HOLDER}}` を「既定値」のCopyright holderへ置き換える。
- 既存の `LICENSE` がある場合は上書きしない。
- README末尾の作者情報の後にライセンス情報を記載する。

## 出力形式

```markdown
# Project Name

何をするためのリポジトリかを簡潔に書く。

## 制作者

[作者名](作者URL)

## ライセンス

[MIT License](LICENSE)
```
