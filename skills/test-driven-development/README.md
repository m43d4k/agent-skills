# test-driven-development

期待する振る舞いを実装より先に明確にし、テストによるフィードバックを設計と実装に利用するAgent Skill。

TDDの形式自体を目的化せず、対象と既存のテスト基盤に応じてテストファーストを適用する。

## 対象

- 観測可能な振る舞いの追加や変更
- 再現可能なバグの修正
- テストで保護しながら行うリファクタリング

## 方針

- テストファーストが有効な場合は、実装前に期待する振る舞いを失敗するテストで示す
- テストが意図した理由で失敗していることを確認する
- 期待する振る舞いを満たすために必要最小限の実装を行う
- 関連するテストが通る状態を維持しながら、必要なリファクタリングを行う
- 実装詳細より、外部から観測可能な振る舞いを優先して検証する
- 変更の影響範囲に応じて検証方法と実行するテストを選ぶ

## 境界

- 純粋なリファクタリングや特性テストのために、人工的な失敗を作らない
- TDDの順序を再現するためだけに既存コードを削除しない
- 無関係なコード、警告、テスト失敗へ作業範囲を広げない
- 既存の問題と今回の変更による問題を区別する
- すべての関数や実装詳細へ個別のテストを要求しない

詳細は [`SKILL.md`](SKILL.md) に記載している。

## 使い方

明示的に使用する場合は、AIエージェントに `test-driven-development` Skillを指定し、実装または修正する振る舞いを伝える。

```text
$test-driven-development を使い、再現可能なバグを修正する。
```
