## 使用条件と試行

- 初見でRealtime AIをキック主体の楽曲に使用した。
- その後もしばらく操作した。
- 仕事での用途を検討するため、複数の音源を書き出して比較した。

## 観測

- Realtime AIはキックを選択的にクリッピングした。
- 一般的なラウドさを得る用途には十分だった。
- クリッパーとリミッターの処理配分は正確だと感じられた。
- Loudnessを操作するだけで使え、短時間でラウドな結果を得られた。
- しばらく操作すると、効率的で合理的な処理である一方、筆者が目標とする音の表現には到達しないと感じられた。
- 人間が同じ処理配分を逐次行うのは困難だと評価した。
- 2Mixの段階でクリッピングが設計されていない音源、またはクリッパーの表現を避けて作られた音源で、ピークを均す用途を試した。
- 複数の書き出しを比較した範囲では、各処理を丹念に設計して最後にリミッターを1段使った結果の方が、仕上がりはよかった。

## 判断と判断履歴

- 初見では、Realtime AIの正確さと手軽さを高く評価した。
- 継続して操作した結果、短時間でラウドにする性能は高いが、筆者が求める表現には届かないという評価が加わった。最初の肯定的評価を撤回したわけではない。
- 仕事で使う場合は、Dynamic AIを有効にしたプリリミッターとして使う方針になった。
- すでに処理を設計した音源で、さらに少しヘッドルームが必要な場合が、その用途に該当する。
- カジュアルな用途では、クリッパーとリミッターの役割分担を短時間で試す手段になる。

## 制約と留保

- AI処理だけで、筆者が求める音の表現まで完成するとは評価していない。
- 手作業より常に仕上がりがよいとは評価していない。
- 製品が提案した値は画面に表示されない。
- 「人間には困難」「正確」という表現は筆者による評価であり、測定による一般的な証明ではない。
