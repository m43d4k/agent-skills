## Loudness ProのRealtime AIを用途別に評価する

初見では、Realtime AIの正確さと手軽さを高く評価した。キック主体の楽曲へ使うとキックを選択的にクリッピングし、Loudnessを操作するだけで一般的なラウドさへ短時間で到達できた。クリッパーとリミッターの処理配分も正確に感じられ、人間が同じ配分を逐次行うのは困難に思えた。

### 操作を続けると評価に条件が加わった

しばらく操作すると、効率的で合理的な処理である一方、筆者が目標とする音の表現には到達しないとも感じられた。2Mixの段階でクリッピングが設計されていない音源や、クリッパーの表現を避けて作られた音源へ適用し、ピークを均した状態で複数を書き出した。しかし、比較した範囲では、各処理を丹念に設計して最後にリミッターを1段使った方が、仕上がりはよかった。

ここで初見の評価が否定されたわけではない。短時間でラウドにする性能と、筆者が求める表現まで仕上げる能力は別の評価軸だった。Realtime AIは前者に優れるが、AI処理だけで後者まで完成するわけではなく、手作業より常によい結果になるともいえない。

### プリリミッターとして使う

試用と比較を経て、仕事ではDynamic AIを有効にしたプリリミッターとして使う方針になった。すでに処理を設計した音源で、さらに少しヘッドルームが必要な場合に、AIの処理配分を利用する。カジュアルな用途なら、クリッパーとリミッターの役割分担を短時間で試す手段にもなる。

「正確」「人間には困難」という部分は筆者の試用に基づく評価であり、測定によって一般化した結果ではない。製品が提案した値も画面には表示されない。Realtime AIの価値は常に最終品質を高めることではなく、条件が合う場合にラウドさへ到達するまでの判断を短縮できる点にある。
