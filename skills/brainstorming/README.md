# brainstorming

実装前の構造化された対話を通じて、曖昧なアイデアを検証済みの設計と仕様へ整理するAgent Skill。

ユーザーが`brainstorming`または`$brainstorming`を明示的に指定した場合、または`multi-agent-brainstorming`から`REJECT`として差し戻された場合だけ使用する。

## 対象

- 新規プロジェクトや主要機能の初期設計
- 完成後の構造的なリファクタリング
- アーキテクチャや技術方針の大きな変更
- 現在の方針からの大きな方向転換

通常の実装、小さな機能追加、バグ修正、調査、Git操作には使用しない。

## 方針

- 質問を始める前に、既存のファイル、ドキュメント、計画、過去の判断を確認する
- 目的、対象ユーザー、制約、成功条件、非目標を一度に一つずつ確認する
- 認識と主要な仮定について合意した後に設計案を提示する
- 複数の設計案とトレードオフを検討し、設計を段階的に検証する
- 主要なリスク、軽減策、残存リスクを確認する
- 判断内容、代替案、採用理由をDecision Logへ記録する
- スキルが有効な間は実装や動作変更を行わない

詳細な規則は[`SKILL.md`](SKILL.md)に記載している。

## 成果物

理解した内容、主要な仮定、リスク、Decision Log、最終設計を文書化する。

プロジェクトに標準的な保存先と形式がある場合はそれに従う。確認できない場合は、対話上で最終設計を提示して保存先と形式を確認する。

## 使い方

AIエージェントに`brainstorming`スキルを明示的に指定し、検討する内容を伝える。

```text
$brainstorming を使い、新しい機能の初期設計を検討する。
```
